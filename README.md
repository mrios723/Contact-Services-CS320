# Contact-Services-CS320
Project Reflection
How can I ensure that my code, program, or software is functional and secure?

I make sure my software is functional by building features directly from the requirements and validating them with testing. I use unit tests to confirm each class and method behaves correctly, and I run regression tests after changes to ensure nothing breaks. To keep things secure, I validate inputs, avoid exposing sensitive data, and follow least-privilege principles (only allowing access to what’s needed). I also use clear error handling so the program fails safely instead of crashing or producing incorrect results.

How do I interpret user needs and incorporate them into a program?

I start by breaking the user requirements into smaller, testable rules (for example: field length limits, required values, unique IDs, and expected behaviors for add/update/delete). Then I translate those rules into acceptance criteria and design my code around them. If something is unclear, I make assumptions based on the requirement intent and document them so the program still aligns with what the user is asking for.

How do I approach designing software?

I design software by keeping it organized, simple, and easy to maintain. I separate responsibilities (data objects vs. services), use clear naming, and keep methods focused on one task. I plan the structure before coding by thinking through how data flows, what validations are required, and how the system should respond to edge cases. I also design with testing in mind so each component can be verified independently.
